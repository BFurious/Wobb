Component Structure, Hooks, and State Management

Overview

This document provides an in-depth explanation of the component structure, React hooks, and state management used in the Wobb Web App redesign project. The application is built using a modular and reusable component architecture, with a focus on maintainability and scalability.

Component Structure

The application is divided into reusable components and pages. Each component is designed to be independent and reusable across the application.

1. Header Component

  • Purpose: Displays the navigation bar, logo, and user actions (e.g., notifications, profile dropdown).
  • Key Features:
    • Responsive design with a mobile menu.
    • Sticky positioning for easy access.
    • Dropdown menu for user actions.
  • Files:
    • Header.tsx: Main component logic.
    • Header.styles.ts: Styled-components for styling.

2. Footer Component

  • Purpose: Displays quick links, customer care options, and social media icons.
  • Key Features:
    • Responsive layout with centered alignment on mobile.
    • Clean and professional design.
  • Files:
    • Footer.tsx: Main component logic.
    • Footer.styles.ts: Styled-components for styling.

3. CampaignCard Component

  • Purpose: Displays campaign details with a flip animation for additional information.
  • Key Features:
    • Flip animation to show more details.
    • Responsive design for mobile, tablet, and desktop.
    • Interactive elements (e.g., buttons, progress bars).
  • Files:
    • CampaignCard.tsx: Main component logic.
    • CampaignCard.styles.ts: Styled-components for styling.

4. FilterBar Component

  • Purpose: Allows users to filter campaigns by eligibility and content type.
  • Key Features:
    • Horizontal scrolling for mobile devices.
    • Active state styling for selected filters.
  • Files:
    • FilterBar.tsx: Main component logic.
    • FilterBar.styles.ts: Styled-components for styling.

5. Pages

  • Home Page: Displays a list of campaigns and featured content.
  • Campaigns Page: Shows detailed information about individual campaigns.
  • Files:
    • Home.tsx: Home page logic.
    • Campaigns.tsx: Campaigns page logic.

Hooks

React hooks are used extensively to manage state and side effects in functional components.

1. useState

  • Purpose: Manages local component state.
  • Examples:
    • Toggle mobile menu visibility in the Header component.
    • Manage flip state in the CampaignCard component.
    • Track active filters in the FilterBar component.
const [isFlipped, setIsFlipped] = useState(false);
const [activeFilter, setActiveFilter] = useState('all');

2. useEffect

  • Purpose: Handles side effects such as data fetching and DOM updates.
  • Examples:
    • Fetch campaign data when the component mounts.
    • Add event listeners for window resizing.
useEffect(() => {
  fetchCampaigns().then((data) => setCampaigns(data));
}, []);

3. useContext

  • Purpose: Manages global state (e.g., user authentication, theme).
  • Examples:
    • Provide theme preferences to all components.
    • Share user authentication state across the app.
const ThemeContext = createContext('light');

const App = () => (
  <ThemeContext.Provider value="dark">
    <Header />
    <MainContent />
    <Footer />
  </ThemeContext.Provider>
);

State Management

State management is handled using a combination of local state (for component-specific data) and global state (for shared data).

1. Local State

  • Managed using the useState hook.
  • Examples:
    • isFlipped in CampaignCard to toggle the flip animation.
    • activeFilter in FilterBar to track the selected filter.

2. Global State

  • Managed using the Context API.
  • Examples:
    • User authentication state.
    • Theme preferences (light/dark mode).

3. State Flow

  • Parent to Child: Props are passed from parent to child components.
  • Child to Parent: Callback functions are used to update parent state.
  • Global State: Context API provides state to all components.

Example: State Flow in CampaignCard

const CampaignCard = ({ campaign }) => {
  const [isFlipped, setIsFlipped] = useState(false);

  return (
    <FlipContainer $isFlipped={isFlipped}>
      <FrontContent>
        <h3>{campaign.brandName}</h3>
        <button onClick={() => setIsFlipped(true)}>View Details</button>
      </FrontContent>
      <BackContent>
        <p>{campaign.description}</p>
        <button onClick={() => setIsFlipped(false)}>Back to Main</button>
      </BackContent>
    </FlipContainer>
  );
};

Design Documentation

Color Scheme

  • Primary Color: #3b82f6 (Blue) - Used for buttons, links, and highlights.
  • Secondary Color: #2dd4bf (Teal) - Used for accents and hover effects.
  • Background: #f9fafb (Light Gray) - Provides a clean and modern look.
  • Text Colors:
    • Primary: #1f2937 (Dark Gray) - For headings and important text.
    • Secondary: #6b7280 (Gray) - For body text and labels.

Typography

  • Font Family: Inter (Sans-serif) - Modern and readable.
  • Font Sizes:
    • Headings: 1.5rem - 2rem
    • Body: 1rem
    • Captions: 0.875rem

Spacing

  • Base Unit: 8px - Used for consistent margins and padding.
  • Card Padding: 1.5rem
  • Section Margins: 2rem

Key Design Decisions

  • Modular Components:
    • Each component is self-contained and reusable.
    • Styles are co-located with components using styled-components.
  • Responsive Design:
    • Mobile-first approach with media queries.
    • Flexbox and CSS Grid for layout.
  • State Management:
    • Local state for component-specific data.
    • Context API for global state.
  • Animations:
    • Flip animations for campaign cards.
    • Smooth transitions for hover effects.

Conclusion

This project demonstrates a clean and scalable architecture using React.js. By leveraging hooks and a modular component structure, the application is easy to maintain and extend. The use of styled-components ensures consistent styling, while the Context API provides a simple solution for global state management.
